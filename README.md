# Trio Release Notes

AI-assisted, source-linked release notes for successful [gordolio/Trio](https://github.com/gordolio/Trio) builds.

Reports use the commit SHA captured in the built app rather than a workflow trigger SHA or release tag. Trio extracts the values from the signed IPA on its private build runner and creates a new `BuildDetails.plist` containing only the commit SHA and build date. Historical artifacts containing an IPA remain supported as a fallback, and temporary files are always deleted.

## Commands

Install and compile with Node.js 24 or later:

```bash
npm ci
npm run build
```

Generate one successful build:

```bash
node dist/src/cli.js generate --run-id 123456789
```

Import successful builds from a rolling month window:

```bash
node dist/src/cli.js backfill --months 2
```

Reconcile recent builds missed by cross-repository dispatch:

```bash
node dist/src/cli.js reconcile --days 30
```

Pass `--checkpoint` in CI to commit and push state, caches, and reports after each successful build. This lets a failed long-running backfill resume without repeating completed AI work.

Pass `--force` to regenerate already processed reports with the current prompt. A repeated build of the same SHA still reuses the report produced by the first build of that SHA.

Backfill includes the nearest successful build before the requested window as its initial comparison baseline. It publishes only builds inside the requested window. Failed workflows, skipped `Build` jobs, and failed `Build` jobs never advance the baseline.

## Configuration

Required secrets in this repository:

| Name | Purpose |
| --- | --- |
| `OPENROUTER_API_KEY` | Structured AI summaries through OpenRouter |
| `TRIO_GITHUB_TOKEN` | Fine-grained token limited to `gordolio/Trio` with Actions, Contents, and Pull Requests read access |

Optional repository variables:

| Name | Default |
| --- | --- |
| `OPENROUTER_MODEL` | `openai/gpt-5-mini` |
| `RELEASE_NOTES_SITE_URL` | `https://github.com/gordolio/Trio-release-notes` |

Trio requires a separate `RELEASE_NOTES_DISPATCH_TOKEN` secret. It must be a fine-grained token limited to this repository with Actions read and write access.
Set the `RELEASE_NOTES_ENABLED` repository variable to `true` in Trio after this repository's workflow has been committed to its default branch. Until then, Trio skips the notification job without affecting builds.

Run the `Generate release notes` workflow manually with `backfill_months` set to `2` for the first import. Commit-range imports are intentionally unsupported; historical reports are always derived from successful build artifacts.

## Generation Pipeline

1. Confirm the exact `4. Build Trio` workflow and one successful `Build` job.
2. Download `release-notes-metadata`, read `BuildDetails.plist`, and remove all temporary files. For historical runs, fall back to `build-artifacts` and read the plist from the IPA.
3. Resolve the SHA in a full Trio checkout and verify ancestry from the preceding successful build.
4. Collect commits, changed files, merge commits, associated fork and upstream pull requests, provenance, and maintenance hotspots.
5. Summarize each PR or logical change through OpenRouter as a short title plus very brief user-facing bullet points, using strict JSON output and cached evidence hashes.
6. Select up to five user-significant changes as build highlights without rewriting their summaries.
7. Reject unknown citations, invalid prose, or omitted changes and render JSON and Markdown deterministically.
8. Commit reports and state, making them immediately available through the public repository and `raw.githubusercontent.com`.

OpenRouter is not given tools or repository access. It receives only bounded source evidence gathered by the generator. Provenance and source URLs are determined locally rather than generated by the model.

## Outputs

- `public/builds.jsonl` (all published builds, newest first, one JSON report per line)
- `public/builds/<short-sha>.json`
- `public/builds/<short-sha>.md`
- `public/builds/<short-sha>.html`
- `public/latest.json`
- `public/latest.md`
- `public/maintenance/<short-sha>.md`

Processing is idempotent by workflow run ID. Repeated successful builds of an already published SHA reuse the existing SHA-keyed report without generating duplicates.
