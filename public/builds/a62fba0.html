Handle NSPersistentHistoryTokenExpiredError during Core Data history fetch by recovering the stored token and retrying. On token-expired error the code now logs the condition, calls recoverFromExpiredHistoryToken() which clears lastToken and retries fetchPersistentHistoryTransactionsAndChanges(); if retry fails it sets lastToken to the store's current persistent history token and calls viewContext.refreshAllObjects() (commit:3c99ae4cfdd1).

This attempts to restore the persistent-history-based change-merge pipeline when the persistent history token has expired by resetting the stored token and replaying available history; if replay fails it jumps to the current token and refreshes in-memory objects, which should prevent the app from silently stopping history-based updates in that error case. The patch notes that this error can happen after long inactivity or DB updates and that it was reproduced on a device, but does not quantify frequency or guarantee all scenarios are covered, so a human review of runtime behavior is recommended if needed. (pr:nightscout/Trio#1247) (commit:3c99ae4cfdd1) (pr:gordolio/Trio#1247)
