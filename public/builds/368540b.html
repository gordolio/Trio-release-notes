Fixes

Fixes an off-by-seconds filter in Stat.StateModel that caused today's per-day meal bucket to be excluded from date-range averages. calculateAveragesForDateRange previously filtered cached per-day entries by testing the bucket's start instant against startDate; the change tests bucket-window overlap (dayStart < endDate && dayEnd > startDate) so a day is included if any moment of it overlaps the requested range (addresses #1181). Separately, the code that built the per-day cache was simplified: the prior grouping + average logic was replaced with a Dictionary keyed by startOfDay using uniquingKeysWith to sum per-day totals, and the cached property/terminology was renamed from dailyAveragesCache to dailyMealTotalsCache (function renamed to calculateAndCacheDailyTotals). Fixes the issue where the Meals stats chart could show zero carbs/fat/protein for the current day because the start-of-day bucket was excluded when the view's startDate was a few seconds past midnight. After this change, the current day's macros should be included in date-range calculations (Sources: pr:gordolio/Trio#1188, commit:5ac5cebefcc4ded3b663c2f35ff750b6f88f7c26, diff:b2879763e027).

When an override duration was 0 (treated as indefinite) the code previously computed end == start, giving the RuleMark zero width and preventing it from rendering. The change makes duration == 0 set the end to Date(timeIntervalSinceNow: 7200) (now + 2 hours); otherwise it uses start.addingTimeInterval(durationAsTimeInterval).

Indefinite overrides that previously did not appear in the Live Activity chart because the rendered bar had zero width should now be visible: the bar is extended to now + 2 hours to match the chart's x-axis domain. Evidence is from the PR description, commit message, and the code diff.
