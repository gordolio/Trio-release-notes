Introduce NightscoutUploadSerializer to serialize, coalesce, and run Nightscout upload runs per NightscoutUploadPipeline (ensures at most one in-flight run per pipeline and one queued follow-up). BaseNightscoutManager upload entry points now call into the serializer; internal upload implementations were renamed to performUpload*() and are executed in the serializer-run context. Removed the prior per-pipeline 2s throttle/Combine machinery. Unit tests for serializer concurrency, queuing, coalescing, reentrancy, and pipeline independence were added.

Fixes duplicate Nightscout treatment uploads caused by overlapping upload runs by ensuring no two runs of the same pipeline overlap. Uploads now start immediately (removed fixed 2s throttle) and queued follow-up requests are coalesced rather than silently dropped. App Intent/awaitable upload semantics are preserved but routed through the serializer. Tests covering the new behavior were added; run the test suite during review.
