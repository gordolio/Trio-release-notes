## Highlights

- **Opt-in Swift oref0 algorithm**
  - Adds an opt-in native Swift port of Trio's oref0 dosing algorithm (IOB, Meal, Autosens, DetermineBasal).
  - Adds a Settings toggle at Algorithm → Additionals → Use Swift Oref to enable the feature.
  - When enabled, Trio uses the Swift implementation for dosing instead of the bundled JavaScript runtime.
  - A green 'Swift Oref' badge is shown in the Loop Status view while the Swift path is active.
  - The JavaScript path remains the default and is unchanged unless you opt in.
  - [View source](https://github.com/nightscout/Trio/pull/1141)
- **Clamp very-low CGM readings to 39 mg/dL**
  - CGM readings below 39 mg/dL are raised to 39 before storage or use.
  - Backfilled CGM data is clamped the same way.
  - The original raw low reading and timestamp are kept in debug logs for diagnostics.
  - User-entered fingerstick (manual) glucose values are not altered.
  - [View source](https://github.com/nightscout/Trio/pull/1159)
- **Glucose fetch follows meal absorption time**
  - Glucose readings used for meal/COB now cover your Max Meal Absorption Time plus 30 minutes.
  - The previous hard 6-hour cap on glucose history is removed so longer MMAT settings are respected.
  - Meal/COB should no longer spike above entered carbs or cliff-drop when MMAT exceeds 6 hours.
  - This behavior is applied consistently for both JS-based and native oref implementations.
  - [View source](https://github.com/nightscout/Trio/pull/1171)
- **dynISF disabled only for high temporary targets**
  - Dynamic sensitivity adjustments (dynISF) will now activate for users whose profile target is above 118.
  - dynISF is only turned off when a temporary target is set at or above the half-basal exercise target (default 160).
  - [View source](https://github.com/nightscout/Trio/pull/1189)
- **Omnipod driver and Live Activity fixes**
  - Legacy OmniKit and OmniBLE drivers were removed; OmnipodKit is now the only Omnipod option.
  - Live Activity now shows active temp targets as a green band and fixes indefinite overrides and widget labels.
  - Watch app recovers from long staleness and refreshes quickly instead of getting stuck.
  - Today's Meals stats now show carbs, fat, and protein correctly instead of zeros.
  - [View source](https://github.com/nightscout/Trio/pull/1212)
