Highlights

Implements scroll-to and transient highlight behavior for Settings search results across searchable settings surfaces. Introduces SettingsSearchHighlight (observable), SearchResultTarget, and two view modifiers (.settingsSearchTarget(label:) and .settingsHighlightScroll()) plus a scrollTargetLabels mapping on SettingItem so search terms can route to visible anchors. The feature is enabled for multiple settings screens (including Algorithm > Dynamic Settings) and includes unit tests and mapping adjustments.

Tapping a Settings search result will navigate to the target screen, auto-scroll to the matching setting row, and briefly highlight it for easier discovery. Multiple settings screens were opted in or updated to apply the new modifiers and several search-index mappings were adjusted to restore expected search terms.

Fixes

Changes TelemetryClient.buildPayload to compute appleHealthEnabled from per-type HealthKit write permissions and add an appleHealthWrites map with per-type booleans for the five Trio-managed sample types.

Telemetry payload semantics changed: appleHealthEnabled will be true if any of the five managed sample types (glucose, insulin, carbs, fat, protein) has write permission, and appleHealthWrites provides the per-type permission snapshot. Downstream telemetry consumers may observe different semantics than the previous all-or-nothing behavior.
