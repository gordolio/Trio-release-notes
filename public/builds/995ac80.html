Highlights

Introduce a LoopGuard actor to serialize loop execution and prevent a race that could start the loop twice; move background-task handling into the loop() async scope (remove shared backgroundTaskID and previous expiration-Task); consolidate loop lifecycle functions into loop()/finalizeLoop(); make calculateLoopInterval accept loopStartDate; and reorder executeLoop to record end/duration/status and only call enactDetermination when settings.closedLoop is true.

Aims to prevent concurrent loop executions by performing the check-and-mark inside a LoopGuard and changes background-task ownership so the loop's async scope begins/ends the background task. The prior expiration handler/Task was removed, so if iOS expires the background task before loop completion, the coroutine completes on a best-effort basis rather than via the former handler. These behavioral changes may affect how loops complete when the app is backgrounded; a human review is recommended to assess runtime and safety impacts in production environments.

Human review recommended.
