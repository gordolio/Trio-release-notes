Core Data performance and stability fixes

  • Background data operations now use fresh Core Data contexts to avoid stale data and reduce races.
  • Database fetches are batched or prefetched to reduce repeated lookups and improve performance.
  • TDD (total daily dose) average calculations run faster due to aggregate queries.
  • Pump-event deduplication now uses timestamp plus type so a bolus and a temp basal at the same time are both kept.
  • Core Data now uses a history-based change feed and FRCs for more consistent background updates.
