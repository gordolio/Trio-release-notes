## Algorithm and Dosing Changes

- **Remove JavaScript oref algorithm**
  - Trio now uses only the Swift oref algorithm.
  - The app setting to run the JavaScript oref was removed.
  - JavaScript algorithm files and related resources were removed from the app bundle.
  - Internal data handling for the algorithm was simplified by removing extra JSON conversion steps.
  - [View source](https://github.com/nightscout/Trio/pull/1317)
- **Use native carb entries instead of JSON**
  - Carb data is passed to the algorithm as structured CarbsEntry objects instead of encoded JSON strings.
  - Carb entries read from storage now retain their stored IDs instead of being dropped by the old JSON round-trip.
  - Carb timestamps keep full precision (no ISO string encode/decode), improving internal date fidelity.
  - The synthetic "additional carbs" entry used for bolus simulation now includes an id and is produced as a CarbsEntry.
