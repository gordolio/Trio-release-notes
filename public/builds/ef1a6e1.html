New Features

Adds an opt-out anonymous telemetry subsystem and UI. Implements a new TelemetryClient and TelemetryAttestor (App Attest-backed) under Services/Telemetry, wires scheduling and immediate send logic (recording cold launches, 24h recurring sends, and immediate send on build change or fresh opt-in), and integrates telemetry consent with existing Crashlytics crash-reporting. Introduces a three-state diagnostics consent (Full / Crash Reports Only / Disabled) with default Full (opt-out on fresh installs), a one-time migration sheet for existing users, Settings and Onboarding UI changes (including a "What's sent" TelemetryPreview), new persistent flags in PropertyPersistentFlags to track telemetry state, and updates PRIVACY_POLICY.md. Also adds the com.apple.developer.devicecheck.appattest-environment entitlement. Existing Crashlytics wiring is left unchanged.

Users will see a new diagnostics choice during onboarding and in Settings → App Diagnostics offering three options: Enable Full Sharing (crashes + anonymous usage telemetry), Crash Reports Only, or Disable Sharing. Existing users who previously completed onboarding are shown a one-time migration sheet on first foreground to choose telemetry consent. A "What's sent" preview lets users inspect the exact JSON payload. Telemetry is sent only when consented and is skipped on unsupported devices (e.g., simulator) and when the attestation registration is forbidden; crash reporting behavior remains controlled by the Crashlytics setting as before. If more detailed review of backend interactions or privacy/legal wording is required, human review may be appropriate.

Fixes

Adds a dedicated NumberFormatter (basalFormatter) to TherapySettingEditorView and uses it to format .unitPerHour values. The formatter sets minimumFractionDigits = 2 and maximumFractionDigits = 3. The .unitPerHour case in the value formatting switch was moved to return basalFormatter.string(from: decimalValue as NSNumber) instead of falling through to the prior decimalValue.description path. Changes are in Trio/Sources/Modules/Onboarding/View/TherapySettingEditorView.swift.

Displayed basal rates for the unitPerHour unit in the onboarding TherapySettingEditorView will now be formatted with a minimum of 2 and a maximum of 3 fractional digits (using the new basalFormatter) instead of using decimalValue.description. This will change the visual rounding/precision for small basal steps; the diff does not show other behavioral changes, but a human review may be desired to confirm no unintended side effects in other code paths.
